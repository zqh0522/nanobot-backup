# A股量化交易系统 - 高手经验

> 来源：东方财富网《OpenClaw实战指南：从零搭建A股自动量化交易系统》（2026-02-27）
> 提取日期：2026-03-13

## 核心理念

- **AI交易 vs 人工交易**：AI的优势在于7×24小时不间断执行、无情绪干扰、精准复现策略
- **工具定位**：OpenClaw不是"预测市场的神器"，而是"执行策略的完美工具"
- **策略先行**：工具只是执行，策略才是核心

## 系统架构：脑手分离

```
大脑（AI分析）：
  ├── 数据采集（QMT + AKShare + 通达信）
  ├── 新闻分析、情绪判断
  └── 生成交易信号

双手（量化平台）：
  ├── 接收信号
  ├── 执行交易
  └── 风险控制（止损线）

数据支撑：
  ├── QMT：实时Level-2行情、历史数据
  ├── 通达信：板块分类
  └── AKShare：财务数据

交互渠道：
  └── 飞书/钉钉/QQ（指令发送与报告推送）
```

## 工作流程

1. AI定时启动
2. 通过QMT获取实时价格、成交量
3. 通过AKShare获取财务指标
4. AI结合技术指标+基本面判断信号
5. 生成标准化交易指令（代码、方向、价格、仓位）
6. 推送至QMT交易接口（或人工确认）
7. 量化平台执行交易
8. 实时监控持仓，自动止损

## 关键成功要素

- ✅ **清晰策略**：先明确"买什么、什么时候买、买多少、什么时候卖"
- ✅ **工具适配**：OpenClaw + QMT + AKShare 组合
- ✅ **风险控制**：凯利准则严格控制仓位 + 自动止损
- ✅ **数据质量**：Level-2实时行情 + 多维度基本面数据
- ✅ **7×24运行**：云服务器部署，不间断监控

## 部署建议

**云服务器配置**：
- 镜像：OpenClaw(Moltbot)镜像
- 实例：内存2GiB及以上
- 地域：中国香港（免ICP，访问国内数据源稳定）
- 端口：18789
- API：阿里云百炼/腾讯云/deepseek等

**数据源配置**：
- 主数据源：QMT（填写安装路径）
- 辅助数据源：通达信板块 + AKShare财务数据
- 实时Level-2行情：需要QMT专业版

## 对当前项目的启发

**akshare + nanobot 方案完全可行**，因为：
1. ✅ 数据源一致：AKShare是官方推荐的辅助数据源
2. ✅ 架构相似：大脑（nanobot AI分析）+ 双手（Python执行）
3. ✅ 零Docker：直接Python脚本，无需复杂部署
4. ✅ 成本更低：不需要QMT专业版（AKShare免费）
5. ✅ 更灵活：完全自己控制代码，可定制

**建议改进**：
- 加入定时任务（cron）实现7×24监控
- 实现信号推送（邮件/钉钉/飞书）
- 添加自动止损逻辑
- 使用凯利公式计算仓位

## 避坑要点

- 不要过度拟合：用滚动窗口回测，避免未来函数
- 注意停牌处理：akshare返回的停牌日数据为NaN
- 复权问题：前复权/后复权选择要一致
- 交易时间：A股只有交易日，避免用日历日
- 涨跌停限制：10%/20%涨跌停会影响策略执行

## 技术栈对比

| 组件 | OpenClaw方案 | 当前方案 |
|------|-------------|----------|
| AI大脑 | OpenClaw | nanobot |
| 数据源 | QMT + AKShare | AKShare |
| 执行平台 | QMT | Python脚本 |
| 部署 | Docker + 云服务器 | 本地Python |
| 成本 | QMT专业版收费 | 免费 |
| 复杂度 | 高（多组件） | 低（单一脚本） |
